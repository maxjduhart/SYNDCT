The main goal of this project is to make (as simple as possible) scripts smaller as a self extracting JavaScript that leverages speed of extracting with the amount of compression done. For client side decompression, this project does a simple split and join of 2 data streams/arrays against a data map. There are no complicated mathematics, no advanced trees to generate, no error checking, no redundancy checks, etc. Not much for the CPU to decode or figure out (even an old 133MHz 486 PC had no issues), there really isn't a simpler way. We get one of the fastest decompressors around (also smallest footprint for the unpacking part, builds in a106 byte unpacker to decompress and run it's self), but only 10 to 30% (some cases up to 66%) in size reduction... You will get better results on www.scriptcompress.com, this is just a cut down (1/3 the coding, 2/3 ripped out), hacked n' slashed version that's easier to use for some people. But this script is a part of that web app, it's about 80% as efficient as the main site.

This script was inspired by the most popular JavaScript packer on the internet, http://dean.edwards.name/packer/.
