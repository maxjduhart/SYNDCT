How Is This Tool Useful?

  1. The biggest use for this is to obfuscate your source code (HTML packed to PHP or PHP compressed to PHP) to make it much harder to read and possibly steal by someone else (also hard for anyone to edit). With normal Base64 encoding, it will make everything 30% bigger, but our method will make your files about half the size or less!
  2. If you wish host and distribute a compressed HTML file (PHP will extract the HTML code before sending out to client, no gain at all really), you may want to consider using our JavaScript packers for HTML code because now your server will send out a compressed file that the client will decompress (saves your server resources and bandwidth) even if the client does not usally know how to decompress due to built in decompressors we supplied (auto extracting and self running JavaScript code to wrap in <script>tags</script>).

Doesn't Base64 Just Make Everything Bigger?

  1. Base64 makes plain text data bigger, but many crazy single characters in GZ files (or any compressed type, ZIP, PNG, GIF, etc) wind up being 2 to 4 bytes (possibly 400 percent bigger rather than only 30% to 50% by Base64) in UTF-8 (default of most browsers) or Unicode compared to just 1 byte in ANSI (archives like ZIP files are actually saved in different file format than web pages, notice that file size and size on hard drive disk conflict, what really is the real size), so we encode to Base64 in ANSI (done in PHP) as all 1 byte characters, now character bytes will not increase at 200% and up (just the 30% from Base64).
  2. When embedding raw data that is compressed, many special characters will break pages and scripts, so Base64 makes it much easier to escape these characters
  3. The point of a packer is to have everything embedded (almost like a self extracting zip archive, but server based).
