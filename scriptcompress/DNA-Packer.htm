  • Most universities avoid using UTF-8/16 charater encoding because some characters are more than 1 byte. This poses an issue when we want to distribute files on a website or share easily on any operating system. Our packer here will make sequences of data smaller using UTF-8 and then pack into a self extracting and automatic running JavaScript that you embed in any HTML page (or web app) in between new <script></script> you create. Now this compressed data can be shared with Linux, iPad, cell phone, Windows or any operating system with a web browser without needing any software to install to run it. -get string to split (DNA, base64 encoded, binary, any unusual text)
-split into bits/pieces array (causes many bits, 3 or 4 times bigger), from 10 chars long to 4 chars
-sort by longest bits (replacing should be done longest first)
-remove bits less than 3 chars long
-remove duplicate bits
-now test each to see how many times found
-calculate savings, if none skip, if limited characters now we have to pick the top (n=#charsUnused)
-sort by best possible savings
-start replacing DNA with results from top to lowest with single unsed characters
-verify more than 2 replacements still possible (sometimes strings overlap others and with other replacements will be canceled out)
-keep track of what replacements were done (sometimes strings overlap others and with other replacements will be canceled out)
-make list of characters used
-map the phrase to character replacer
-now repeat steps using smaller bits/pieces and loop until we reach desired amout
-now pack to self extracting JavaScript snippet
-hope to get 80% reduction in size
-implement to C++ because it will be CPU intensive!
    SOME INTERESTING FINDINGS:

-even making bits/pieces 66 characters long we are able to find multiple repetitions
